# Contributing to Discord Summary Bot

Thank you for your interest in contributing to Discord Summary Bot!

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or bug fix
4. Make your changes
5. Test thoroughly
6. Submit a pull request

## Development Setup

Follow the installation instructions in the README, then:

```bash
npm run dev  # Start bot with auto-reload
```

## Code Guidelines

- Use ES6+ features and modern JavaScript
- Follow the existing code style and structure
- Add comments for complex logic
- Keep functions focused and single-purpose
- Handle errors gracefully with proper logging

## Testing Your Changes

Before submitting a PR:
1. Test the bot in a Discord server
2. Verify rate limiting works correctly
3. Test with different LLM providers if possible
4. Check that error handling works as expected

## Submitting Changes

1. Ensure your code follows the project structure
2. Update documentation if needed
3. Describe your changes clearly in the PR description
4. Reference any related issues

## Feature Requests

Have an idea? Open an issue with:
- Clear description of the feature
- Use case and benefits
- Any implementation suggestions

## Bug Reports

Found a bug? Open an issue with:
- Steps to reproduce
- Expected vs actual behavior
- Error messages and logs
- Your environment (Node version, OS, etc.)

## Questions

Feel free to open an issue for questions or discussions.

Thank you for contributing!
